Crossplane 2.3.0 shipped on May 26, 2026, led by a change to how `crossplane render` produces output during local development.

## Render parity

`crossplane render` now drives the actual composite reconciler instead of a parallel re-implementation, so what you see locally matches what the control plane will reconcile in-cluster. The command pulls the Crossplane container image by default to do this. The long-standing gap between dry-run rendering and real reconciliation is the kind of thing that turns a clean local diff into a surprise in production.

## Per-resource reconciliation control

Two annotations move reconciliation tuning from the controller flag down to individual resources. `crossplane.io/poll-interval` overrides the controller-wide poll for a single resource (for example `24h` on a resource that rarely drifts), and `crossplane.io/reconcile-requested-at` forces an immediate reconcile whenever its value changes. This is stable for composite resources; managed resources need a provider built against the updated runtime to honour it.

## Provider deletion protection

A new alpha flag, `--enable-provider-deletion-protection`, has Crossplane create `ClusterUsage` resources that block accidental deletion of Provider packages through the existing Usage webhook. The release also adds `commonAnnotations` propagation from Compositions to composed resources and lets a CompositeResourceDefinition expose a scale subresource.
