Kubescape 4.0 dropped at KubeCon Europe in Amsterdam, and it's doing something no other open source Kubernetes security tool has attempted: scanning the security posture of AI agents running inside your clusters.

The release promotes two features to GA. Runtime Threat Detection now uses CEL-based detection rules that learn normal workload behavior through Application Profiles, then alert on deviations. ARMO, the project's primary maintainer, claims this cuts CVE noise by over 95%. That's a bold number, but the approach is sound. Instead of drowning you in every known vulnerability for every image, it focuses on what's actually executing at runtime.

Kubescape Storage also hits GA. It uses the Kubernetes Aggregated API to store SBOMs, vulnerability manifests, and Application Profiles in a dedicated layer, keeping that metadata out of etcd. For large clusters, that's a meaningful architectural decision.

## The AI Agent Angle

Here's where it gets interesting. Kubescape 4.0 ships 15 Rego-based controls covering 42 security-critical configuration points in KAgent's CRDs. These check for things like empty security contexts on default deployments, missing NetworkPolicies, and controllers with over-privileged namespace-wide watch permissions.

If you're running agentic AI workloads on Kubernetes, these agents often get deep infrastructure access. That's a new attack surface that most security tooling simply ignores. Kubescape is the first project to systematically target it.

There's also a KAgent-native plug-in that lets AI assistants query your cluster's security posture directly. They can inspect vulnerability manifests, review RBAC configurations, and analyze container runtime behavior through ApplicationProfiles and NetworkNeighborhoods.

## Cleaner Architecture

The 4.0 release removes the old host-sensor DaemonSet and folds those capabilities into the node-agent. One agent per node, direct API communication between core microservices. Less privilege escalation surface, easier auditing.

Compliance coverage now includes CIS Benchmark 1.12 for vanilla Kubernetes and CIS Benchmark 1.8 for EKS and AKS, alongside existing NSA-CISA and MITRE ATT&CK framework support.

Ben Hirschberg, Kubescape core maintainer and ARMO CTO, led the release. New maintainer Amir Malka joined the project, while David Wertenteil and Craig Box moved to emeritus status.

If you're already running Kubescape, the upgrade path is straightforward since it's a Helm chart update. If you're not, and you've got AI agents anywhere near your clusters, this is worth a look. The 42-point KAgent security scan alone will probably surface something you didn't know about.
