Broadcom filed Velero's CNCF Sandbox application in February and made the announcement official at KubeCon EU in Amsterdam. The project now sits under vendor-neutral governance, which matters because Velero is how most Kubernetes teams handle backup and disaster recovery.

Kubernetes doesn't ship with cluster-level backup. That's always been Velero's niche. It operates at the API layer rather than poking at etcd directly, which means backups stay portable across providers and distributions. Scheduled backups via cron expressions, CSI snapshot integration, file system backup through Restic and Kopia, pre/post hooks for custom commands, cross-cluster migration. It's the Swiss Army knife of Kubernetes data protection.

The maintainer list already spans three companies: Broadcom, Red Hat, and Microsoft. That's not window dressing. Multi-vendor maintainership before a CNCF donation is unusual and signals the project won't drift toward any single company's roadmap.

Chris Aniszczyk, CNCF's CTO, pointed to vendor-neutral governance as the key benefit. Fair enough. But what matters more to practitioners is what happens next. Velero's been Apache 2.0 licensed for years, so the code was already open. The real change is structural. CNCF Sandbox projects get legal support, mentorship, and a clear path toward incubation and graduation. Contributions from outside the current maintainer circle should get easier.

Here's the practical question for platform teams: does this change anything today? Not really. Your Helm charts still work. Your backup schedules still run. The project isn't being forked or rewritten.

But if you've been running Velero and quietly worrying about Broadcom's long-term commitment to open source tooling after the VMware acquisition, this is the answer. They're not pulling a HashiCorp. They're moving in the opposite direction.

The Sandbox application was presented to TAG Operational Resilience on February 4, with Nikhita Raghunath from Broadcom leading the submission. Velero integrates with Prometheus for metrics, ships an official Helm chart, and plays nicely with Rook, Longhorn, and cert-manager.

One thing to watch: Sandbox is just the starting line. The project needs to demonstrate growing adoption and contributor diversity to move toward incubation. Given that Velero is already a default choice for most Kubernetes backup workflows, the bigger question is whether CNCF governance actually accelerates development or just adds process. We'll find out over the next year.
