Helm Dashboard was a web UI for inspecting and managing Helm releases in a Kubernetes cluster, originally built by Komodor. It listed installed releases, showed manifest diffs between revisions, let you browse the actual Kubernetes resources a chart produced, and exposed rollback and upgrade buttons on top of the Helm SDK.

The project is abandoned. The repository has not received meaningful updates since Komodor stopped investing in it, and it is not maintained by the Helm project itself — despite the name, it was never an official Helm subproject. It still works against older Helm releases but does not track current Helm 3.x changes.

If you want a GUI for Helm today, look at k9s (TUI with Helm plugin), Headlamp, or the Helm integration in whatever cluster dashboard you already run. For GitOps-driven release management, Argo CD and Flux have replaced the "click to upgrade" workflow Helm Dashboard was targeting.
